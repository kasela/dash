Security

Your data security is our priority

We take security seriously at every layer — from how your files are stored to how your dashboards are shared.

Encryption in transit & at rest

All data transmitted between your browser and DashAI is encrypted via TLS 1.2+. Uploaded datasets and account data are encrypted at rest using AES-256.

Access controls

Your dashboards and datasets are private by default. Sharing requires an explicit action from you. Share links use cryptographically random UUID tokens and can be revoked at any time.

Authentication security

Passwords are hashed using Django's PBKDF2-SHA256 with per-user salts. We support secure password reset flows and session management with CSRF protection on all forms.

Infrastructure

DashAI runs on reputable cloud infrastructure with automated backups, network-level firewalls, and regular security patches applied to all system components.

API key security

API keys are hashed before storage and displayed only once at creation time. Keys can be revoked instantly from your account settings. Treat them like passwords.

Data isolation

Each workspace's data is logically isolated. Queries are scoped to the authenticated user's workspace, preventing cross-user data access through our API or UI.

Responsible disclosure

If you discover a security vulnerability in DashAI, please report it responsibly. We will investigate all reports and work quickly to fix confirmed issues. We ask that you:

  • Do not access or modify other users' data
  • Give us reasonable time to fix issues before public disclosure
  • Provide enough detail to reproduce the issue
